# Agent D.O.J.O. — Package Build & Installer Instructions

## Overview

This document describes how to build a deployable package for the Agent D.O.J.O. platform, including the macOS `.pkg` installer and menu bar app.

---

## Prerequisites (Dev Machine)

- macOS 13+ with Xcode Command Line Tools (`xcode-select --install`)
- Node.js 22+ (`brew install node@22`)
- npm (comes with Node)
- Swift compiler (comes with Xcode CLI tools)
- `create-dmg` (optional, for DMG packaging: `brew install create-dmg`)

---

## Step 1: Build Everything

```bash
cd cornerpin-platform
bash deploy/build-package.sh
```

This script:
1. Builds `packages/shared` (TypeScript → JS)
2. Builds `packages/server` (TypeScript → JS)
3. Builds `packages/dashboard` (Vite → static HTML/CSS/JS)
4. Builds `watchdog` (TypeScript → JS)
5. Compiles `menubar/DojoMenuBar.swift` → `DOJO.app` (native macOS menu bar app)
6. Assembles everything into `deploy/dist/dojo-platform/`
7. Creates `deploy/dist/dojo-platform.zip`

Output: `deploy/dist/dojo-platform.zip` (~XXmb)

## Step 3: What install.sh Does

When a user runs `bash install.sh` on a fresh Mac:

1. **Preflight**: Checks macOS, checks RAM (warns if <8GB)
2. **Homebrew**: Installs if missing
3. **Node.js 22**: Installs via brew if missing or outdated
4. **Directory structure**: Creates `~/.dojo/{data,logs,prompts,uploads,techniques}`
5. **Platform files**: Copies `platform/` to `~/.dojo/platform/`
6. **npm ci**: Installs production Node dependencies
7. **Secrets**: Generates fresh JWT secret, creates `~/.dojo/secrets.yaml`
8. **Icon**: Copies `dojologo.pdf` to `~/.dojo/`
9. **Menu bar app**: Copies `DOJO.app` to `/Applications/`, adds to login items
10. **Launchd plists**: Generates `com.dojo.platform.plist` and `com.dojo.watchdog.plist` in `~/Library/LaunchAgents/` with correct absolute paths for this machine
11. **Starts services**: Loads both launchd plists
12. **Health check**: Polls `/api/health` for up to 30 seconds
13. **Opens browser**: `http://localhost:3000` → setup wizard
14. **Launches menu bar**: Opens `DOJO.app`

## Step 4: Building a .pkg Installer (Future Enhancement)

To create a standard macOS `.pkg` installer instead of a zip+script:

### Tools needed
- `pkgbuild` (built into Xcode CLI tools)
- `productbuild` (built into Xcode CLI tools)

### Approach

```bash
# 1. Build everything first
bash deploy/build-package.sh

# 2. Create a component package from the dist directory
pkgbuild \
  --root deploy/dist/dojo-platform \
  --identifier com.dojo.platform \
  --version 1.0 \
  --install-location /tmp/dojo-install \
  --scripts deploy/pkg-scripts \
  deploy/dist/dojo-component.pkg

# 3. Create the final product package with welcome/license screens
productbuild \
  --distribution deploy/distribution.xml \
  --resources deploy/pkg-resources \
  --package-path deploy/dist \
  deploy/dist/Agent-DOJO-Installer.pkg
```

### pkg-scripts/ directory
Create `deploy/pkg-scripts/postinstall`:
```bash
#!/bin/bash
# Run the actual installer from the temp location
cd /tmp/dojo-install
bash install.sh
# Clean up temp
rm -rf /tmp/dojo-install
```

### distribution.xml
```xml
<?xml version="1.0" encoding="utf-8"?>
<installer-gui-script minSpecVersion="2">
    <title>Agent D.O.J.O.</title>
    <welcome file="welcome.html"/>
    <license file="license.html"/>
    <conclusion file="conclusion.html"/>
    <options customize="never" require-scripts="true"/>
    <domains enable_anywhere="true"/>
    <pkg-ref id="com.dojo.platform"/>
    <choices-outline>
        <line choice="default">
            <line choice="com.dojo.platform"/>
        </line>
    </choices-outline>
    <choice id="default"/>
    <choice id="com.dojo.platform" visible="false">
        <pkg-ref id="com.dojo.platform"/>
    </choice>
    <pkg-ref id="com.dojo.platform" version="1.0" onConclusion="none">dojo-component.pkg</pkg-ref>
</installer-gui-script>
```

### pkg-resources/ directory
- `welcome.html` — "Welcome to Agent D.O.J.O." with logo
- `license.html` — MIT license or terms
- `conclusion.html` — "Installation complete! The setup wizard has opened in your browser."
- `background.png` — Installer background image (optional)

### Signing (for Gatekeeper)
To distribute outside the App Store without "unidentified developer" warnings:
```bash
productsign --sign "Developer ID Installer: Your Name" \
  deploy/dist/Agent-DOJO-Installer.pkg \
  deploy/dist/Agent-DOJO-Installer-signed.pkg
```
Requires an Apple Developer account ($99/year).

## Step 5: What uninstall.sh Does

1. Confirms with user
2. Stops launchd services (platform + watchdog)
3. Quits DOJO.app menu bar app
4. Backs up `~/.dojo/data/`, secrets, prompts, techniques to `~/dojo-backup-{timestamp}/`
5. Removes launchd plists from `~/Library/LaunchAgents/`
6. Removes `DOJO.app` from `/Applications/`
7. Removes login item
8. Removes `~/.dojo/` entirely

## Architecture Notes

### Production Mode

In production (`NODE_ENV=production`), the server:
- Serves the built dashboard static files from `packages/dashboard/dist/`
- Handles SPA routing (all non-API routes serve `index.html`)
- Both dashboard and API run on port 3001 (no Vite dev server)
- The tunnel (if enabled) points to port 3001

In development (`npm run dev`):
- Vite dev server on port 3000 (with HMR)
- API server on port 3001
- Vite proxies `/api` to port 3001

### Menu Bar App

- Written in Swift, ~150 lines
- Compiled to native ARM64 binary (macOS 13+)
- `LSUIElement: true` — menu bar only, no dock icon
- Polls `/api/health` every 10 seconds for status
- Can start/stop the server via `~/.dojo/scripts/`
- Added to login items during install (auto-starts on boot)
- Icon: `dojologo.pdf` loaded as template image (auto light/dark mode)

### Launchd Services

- `com.dojo.platform` — Main Node.js server, `KeepAlive: true`, throttle 10s
- `com.dojo.watchdog` — Health monitor, `KeepAlive: true`, throttle 30s
- Both log to `~/.dojo/logs/`
- Plists generated by install.sh with machine-specific paths

### Data Location

All user data in `~/.dojo/`:
- `data/dojo.db` — SQLite database
- `secrets.yaml` — API keys, JWT secret, password hash
- `prompts/` — Customizable SOUL.md, PM-SOUL.md, TRAINER-SOUL.md, USER.md
- `techniques/` — Learned technique packages
- `uploads/` — User-uploaded files (auto-cleaned after 7 days)
- `logs/` — Application and service logs
- `platform/` — The server and dashboard code
- `watchdog/` — Health monitor code
- `scripts/` — start/stop/status/backup/uninstall
